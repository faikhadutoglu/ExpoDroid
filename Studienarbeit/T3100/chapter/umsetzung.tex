\section{Mechanische Konstruktion}
\subsection{CAD-Modellierung}

Vor der praktischen Umsetzung des Roboters wurde zunächst ein digitales CAD-Modell erstellt. Ziel dieser Modellierung war es, bereits in einer frühen Projektphase einen groben, jedoch strukturierten Bauplan zu entwickeln, der sowohl die äußere Gestaltung als auch den grundsätzlichen mechanischen Aufbau des Systems abbildet. Dadurch konnten gestalterische, konstruktive und funktionale Aspekte frühzeitig bewertet und iterativ angepasst werden.

Die CAD-Modellierung erfolgte mit einer browserbasierten, cloudgestützten CAD-Anwendung \textcolor{red}{(Onshape)}. Ein wesentlicher Vorteil dieses Werkzeugs liegt in der Möglichkeit zur kollaborativen Zusammenarbeit, da mehrere Projektmitglieder zeitgleich an einem Modell arbeiten können. Dies erleichtert die Arbeitsteilung innerhalb eines studentischen Projekts erheblich und ermöglicht eine kontinuierliche Weiterentwicklung des Entwurfs unabhängig vom verwendeten Endgerät oder Standort.

Der CAD-Entwurf diente primär als konzeptionelle Grundlage für den späteren physischen Aufbau des Roboters. Dabei lag der Fokus weniger auf einer vollständig fertigungsgerechten Detailkonstruktion, sondern vielmehr auf der Definition der grundlegenden Struktur, der Proportionen sowie der Anordnung zentraler Komponenten. Auf diese Weise konnten sowohl stilistische Aspekte des Designs als auch technische Randbedingungen untersucht und aufeinander abgestimmt werden.

Ein weiterer wichtiger Zweck der CAD-Modellierung bestand in der Abschätzung von Materialbedarf, Bauraum und Bauteilpositionierungen. Insbesondere die Dimensionierung der tragenden Struktur sowie die Platzierung von Antriebskomponenten, Elektronik und Verkleidungselementen konnten im digitalen Modell überprüft und optimiert werden. Darüber hinaus ließ sich bereits vor dem Bau einschätzen, welche Fertigungsschritte erforderlich sind und welche Bauteile mit den zur Verfügung stehenden Mitteln realisierbar sind.

Die Verwendung eines digitalen Modells vor Beginn der mechanischen Fertigung trägt wesentlich zur Vermeidung von Konstruktionsfehlern bei. Größere Änderungen lassen sich im CAD-Modell mit vergleichsweise geringem Aufwand umsetzen, während nachträgliche Anpassungen am physischen Aufbau häufig mit erheblichem Zeit- und Kostenaufwand verbunden sind. Durch die vorgelagerte Modellierung konnten somit sowohl der Materialverbrauch als auch das Risiko eines vollständigen Neubaus reduziert werden.

\subsection{Montage und Nachbearbeitung}
Auf Basis des CAD-Modells wurde schließlich die Entscheidung für den Werkstoff Holz als primäres Konstruktionsmaterial getroffen. Holz bietet im Vergleich zu Metall ein geringeres Gewicht bei gleichzeitig ausreichender Stabilität für den geplanten Einsatzzweck. Zudem lässt es sich mit einfachen Werkzeugen bearbeiten und eignet sich daher besonders für die eigenständige Fertigung im Rahmen eines studentischen Projekts. Im Vergleich zu Kunststoffbauteilen aus dem 3D-Druck war das verfügbare Druckvolumen nicht ausreichend, um größere tragende Strukturelemente am Stück herzustellen.

Der aktuelle Stand der mechanischen Konstruktion basiert auf einer groben Holzstruktur, die die grundlegende Form und Tragfähigkeit des Roboters vorgibt. Diese Struktur dient als funktionales Grundgerüst und soll in einem weiteren Schritt durch zusätzliche Verkleidungselemente ergänzt werden. Geplant ist, diese Verkleidung teilweise mittels additiver Fertigung herzustellen und auf die Holzstruktur aufzuschrauben, um sowohl die Optik als auch den Schutz der inneren Komponenten zu verbessern.

\section{Motorsteuerung und Bewegungsalgorithmen}


\subsection{Hardwareanbindung der Motortreiber}
Die Motoransteuerung erfolgt über zwei BTS7960 H-Brücken-Module, die eine bidirektionale Steuerung von Gleichstrommotoren mit hohen Strömen ermöglichen. Jedes Modul verfügt über zwei PWM-Eingänge:
\begin{itemize}
	\item \text{RPWM (Right PWM)}: Steuerung der Vorwärtsdrehung
	\item \text{LPWM (Left PWM)}: Steuerung der Rückwärtsdrehung
\end{itemize}

Die Pin-Konfiguration im System:
\begin{lstlisting}[language=C++]
	// Modul 1 (Linker Motor)
	PIN_BTS1_RPWM  // Vorwaerts-PWM
	PIN_BTS1_LPWM  // Rueckwaerts-PWM
	
	// Modul 2 (Rechter Motor)
	PIN_BTS2_RPWM  // Vorwaerts-PWM
	PIN_BTS2_LPWM  // Rueckwaerts-PWM
\end{lstlisting}


\subsection{Implementierung der Motorsteuerung}


Die Initialisierungsphase konfiguriert alle relevanten Pins als Ausgänge und bereitet die PWM-Generierung vor:
\begin{lstlisting}[language=C++]
	void actuatorControllerInit() {
		// Modul 1 PWM-Pins
		pinMode(PIN_BTS1_RPWM, OUTPUT);
		pinMode(PIN_BTS1_LPWM, OUTPUT);
		
		// Modul 2 PWM-Pins
		pinMode(PIN_BTS2_RPWM, OUTPUT);
		pinMode(PIN_BTS2_LPWM, OUTPUT);
		
		moduleIsInitialised = true;
	}
\end{lstlisting}


Die Hauptsteuerungslogik in der Funktion \texttt{actuatorControllerForwardSignals()} implementiert eine kontinuierliche Signal-zu-Bewegungs-Transformation.

\textbf{1. Signalerfassung}
\begin{lstlisting}[language=C++]
	pulseLength_us = pulseIn(servoSignalPinsRC[i], HIGH,
	TIMEOUT_SERVO_SIGNAL_HIGH_US);
\end{lstlisting}
Die Funktion liest die Pulsbreite der RC-Signale ein. Bei einem Timeout (kein Signal) wird \texttt{0} zurückgegeben, was als Fehlerfall behandelt wird.

\textbf{2. Kanalspeicherung}
\begin{lstlisting}[language=C++]
	if (i == ENGINE) {
		lastSteeringPulse = pulseLength_us;
	} else if (i == AILERON) {
		lastSpeedPulse = pulseLength_us;
	}
\end{lstlisting}
Die relevanten Steuerkanäle werden zur weiteren Verarbeitung zwischengespeichert.

\textbf{3. Motorrichtungslogik}
Die bidirektionale Motoransteuerung erfolgt durch exklusive Aktivierung einer der beiden PWM-Leitungen:

\textbf{Linker Motor -- Vorwärtsfahrt} ($leftMotorSpeed > 0$):
\begin{lstlisting}[language=C++]
	analogWrite(PIN_BTS1_RPWM, abs(leftMotorSpeed));
	analogWrite(PIN_BTS1_LPWM, 0);
\end{lstlisting}

\textbf{Linker Motor -- Rückwärtsfahrt} ($leftMotorSpeed < 0$):
\begin{lstlisting}[language=C++]
	analogWrite(PIN_BTS1_RPWM, 0);
	analogWrite(PIN_BTS1_LPWM, abs(leftMotorSpeed));
\end{lstlisting}

\textbf{Linker Motor -- Stopp} ($leftMotorSpeed = 0$):
\begin{lstlisting}[language=C++]
	analogWrite(PIN_BTS1_RPWM, 0);
	analogWrite(PIN_BTS1_LPWM, 0);
\end{lstlisting}

Identische Logik wird für den rechten Motor (Modul~2) implementiert.


\subsubsection{Bewegungsprofile}
Das entwickelte System unterstützt verschiedene Bewegungsprofile:

\paragraph{Geradeausfahrt}
\begin{itemize}
	\item Bedingung: $steering = 0$
	\item Verhalten: $leftMotorSpeed = rightMotorSpeed = baseSpeed$
	\item Ergebnis: Beide Motoren laufen synchron
\end{itemize}

\paragraph{Kurvenfahrt}
\begin{itemize}
	\item Bedingung: $steering \neq 0$
	\item Verhalten: Geschwindigkeitsdifferenz zwischen den Motoren
	\item Ergebnis: Fahrzeug vollzieht eine Kurve mit Radius abhängig vom Lenkausschlag
\end{itemize}

\subsubsection{Sicherheitsmechanismen}

\paragraph{Timeout-Behandlung}
Ein Timeout beim Signaleinlesen (Rückgabewert \texttt{0}) verhindert die Aktualisierung der Motorwerte, sodass die letzten gültigen Werte beibehalten werden.

\paragraph{Wertebereichsbegrenzung}
Die \texttt{constrain()}-Funktion stellt sicher, dass PWM-Werte niemals die physikalischen Grenzen der Hardware überschreiten.

\paragraph{Spannungsreduktion}
Der Batterie-Korrekturfaktor (\num{0.91}) schützt vor Überspannung und verlängert die Batterielebensdauer durch gemäßigte Spitzenlast.

\subsubsection{Performance-Charakteristiken}
Die implementierte Steuerung zeichnet sich aus durch:
\begin{itemize}
	\item \textbf{Reaktionszeit}: $< \SI{20}{\milli\second}$ durch kontinuierliche Polling-Schleife
	\item \textbf{Präzision}: 8-Bit PWM-Auflösung (256 Stufen pro Richtung)
	\item \textbf{Stabilität}: Keine Oszillationen durch direkte Ansteuerung ohne Regelkreis
	\item \textbf{Intuitivität}: Natürliches Fahrverhalten durch proportionale Lenkcharakteristik
\end{itemize}
